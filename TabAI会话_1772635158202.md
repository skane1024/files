# C++11 算子动态 Shape 规则优化与存储方案

本文档旨在针对算子动态 Shape 变化的数据结构进行优化，并提供一种基于 C++11 和 JsonCpp 的高效存储与检索方案。

## 1. 数据结构优化建议

原始定义中存在较多重复的 `ShapeRule` 引用，且使用传统的 `typedef enum` 容易导致命名冲突。优化方向如下：

*   **类型安全**：使用 `enum class` 替代 `typedef enum`，避免枚举成员污染全局命名空间。
*   **空间效率**：利用 `union`（联合体）将不同算子的规则组合在一起。由于这些结构体成员多为基础类型（POD），`union` 可以显著降低内存占用。
*   **结构精简**：将仅包含单一输入 Shape 的算子（如 Softmax, RmsNorm, Rope）统一归类为通用规则。

### 优化后的代码定义

```cpp
#include <string>
#include <unordered_map>
#include <vector>

// 使用 enum class 增强类型安全
enum class OpType {
    MATMUL,
    ELEMENTWISE,
    SOFTMAX,
    COPY,
    RMS_NORM,
    ROPE
};

enum class ShapeDimRule {
    FIXED,
    SEQ_LENGTH,
    CACHE_LENGTH
};

enum class DDROffsetRule {
    FIXED,
    BLOCK_INDEX_BASED
};

// 基础 Shape 规则
struct ShapeRule {
    ShapeDimRule n_rule = ShapeDimRule::FIXED;
    ShapeDimRule h_rule = ShapeDimRule::FIXED;
    ShapeDimRule w_rule = ShapeDimRule::FIXED;
    ShapeDimRule c_rule = ShapeDimRule::FIXED;
};

// 算子特定参数结构
struct MatmulRule {
    ShapeRule in_fm0;
    ShapeRule in_fm1;
    ShapeRule out_fm;
    DDROffsetRule fm1_offset_rule;
    int block_index;
};

struct CopyRule {
    ShapeRule in_shape;
    ShapeRule out_shape;
    DDROffsetRule store_offset_rule;
    int block_index;
};

// 统一规则包装器（使用 Union 节省空间）
struct OpDynamicRule {
    OpType type;
    union {
        MatmulRule matmul;
        CopyRule copy;
        ShapeRule generic_shape; // 用于单输入/输出算子
    } data;
};
```

## 2. 高效存储与检索

在 C++11 中，推荐使用 `std::unordered_map` 来建立算子名称（Name）到规则（Rule）的映射。其平均查找复杂度为 $O(1)$。

### 规则管理器封装

```cpp
class OpRuleManager {
public:
    // 添加规则
    void addRule(const std::string& name, const OpDynamicRule& rule) {
        rules_[name] = rule;
    }

    // 通过名称获取规则
    bool getRule(const std::string& name, OpDynamicRule& out_rule) const {
        auto it = rules_.find(name);
        if (it != rules_.end()) {
            out_rule = it->second;
            return true;
        }
        return false;
    }

private:
    std::unordered_map<std::string, OpDynamicRule> rules_;
};
```

## 3. 基于 JsonCpp 的序列化方案

为了实现规则的持久化存储（存入文件并重新解析），建议将 `unordered_map` 序列化为 JSON 格式。

### 序列化逻辑实现

```cpp
#include <json/json.h>

// 辅助函数：ShapeRule 转 Json
Json::Value encodeShapeRule(const ShapeRule& rule) {
    Json::Value val;
    val["n"] = static_cast<int>(rule.n_rule);
    val["h"] = static_cast<int>(rule.h_rule);
    val["w"] = static_cast<int>(rule.w_rule);
    val["c"] = static_cast<int>(rule.c_rule);
    return val;
}

// 整体序列化函数
std::string serializeRules(const std::unordered_map<std::string, OpDynamicRule>& rules) {
    Json::Value root;
    for (auto const& it : rules) {
        Json::Value opJson;
        opJson["type"] = static_cast<int>(it.second.type);
        
        switch (it.second.type) {
            case OpType::MATMUL:
                opJson["in0"] = encodeShapeRule(it.second.data.matmul.in_fm0);
                opJson["in1"] = encodeShapeRule(it.second.data.matmul.in_fm1);
                opJson["out"] = encodeShapeRule(it.second.data.matmul.out_fm);
                opJson["offset"] = static_cast<int>(it.second.data.matmul.fm1_offset_rule);
                opJson["b_idx"] = it.second.data.matmul.block_index;
                break;
            case OpType::COPY:
                opJson["in"] = encodeShapeRule(it.second.data.copy.in_shape);
                opJson["out"] = encodeShapeRule(it.second.data.copy.out_shape);
                opJson["offset"] = static_cast<int>(it.second.data.copy.store_offset_rule);
                opJson["b_idx"] = it.second.data.copy.block_index;
                break;
            default:
                opJson["shape"] = encodeShapeRule(it.second.data.generic_shape);
                break;
        }
        root[it.first] = opJson;
    }
    
    Json::StreamWriterBuilder writer;
    return Json::writeString(writer, root);
}
```

## 4. 方案总结

### 优点
1.  **简洁性**：不引入 Boost 等重型库，仅依赖标准的 C++11 和轻量级的 JsonCpp。
2.  **高性能**：使用 `unordered_map` 保证了在大规模算子模型下的查找效率；使用 `union` 优化了内存布局。
3.  **可维护性**：通过将枚举转为整数或字符串存储在 JSON 中，方便开发者手动查阅和修改配置文件。

### 进一步优化建议
*   **二进制存储**：如果模型规模极大（数万个算子），JSON 解析可能变慢。此时可考虑将 `OpDynamicRule` 数组直接以二进制形式写入文件（Dump），读取时直接 `memcpy` 到内存。
*   **默认构造**：在结构体中使用 C++11 的成员初始化（如 `int block_index = 0;`），防止内存脏数据导致逻辑异常。
